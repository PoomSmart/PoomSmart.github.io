Which does what?

Emoji10 (all variants) adds latest emojis on your emoji keyboard. It may need following other tweaks depending on iOS version you are at to make everything works as intended.

EmojiAttributes

This tweak helps display emoji characters properly. Otherwise, we may see "?", "U", square or just blank space instead of actual emojis.

EmojiLocalization

This tweak adds missing localized strings on emoji keyboard.

EmojiResources

This tweak adds missing category icons on emoji keyboard.

EmojiFontManager

This tweak allows users to install correct emoji font without filesystem modification.

EmojiDarkStyle

This tweak is optional. It enables dark variants of emoji category icons available on iOS 10.2+.

Read first about Emoji10

Emoji has been integrated into iOS as a keyboard. All emojis you see on the keyboard are defined as lists within the system and vary across iOS versions. Newer OS versions usually include more emojis so users at lower version won't be able to use unless they update their iOS.

All emoji images that you see are packed as a font, named AppleColorEmoji. In other words, an emoji font is crucial in order to display emojis correctly. Of course, the font gets updated too to include images of new emojis.

Emoji10 tweak will override the original emoji list with the up-to-date one. It means your emoji keyboard will have more emojis, also latest. In addition, Emoji10 requires an up-to-date font that contains all emoji images needed. Without new font installed, users are going to see blank emojis - in other words, new emojis don't show up.

How can I get the correct emoji font?

Assuming you have already added my Cydia repository: poomsmart.github.io/repo

It is recommended to backup the original fonts before having a new one

Option #1: BytaFont (iOS < 10)

Installation

  1. Make sure you have enough space on system partition (over 100 MB is recommended)
    2.
  2. Install BytaFont app (BytaFont 3 if iOS 9, BytaFont 2 if iOS 7-8, BytaFont if iOS 6)
  3. Add this Cydia repository: https://vxbakerxv.github.io/repo
  4. Install Emoji iOS 11 font
  5. Open BytaFont app, go to Swap Mode > Emoji and tap Emoji10 to enable the font, respring after that

Uninstallation

  1. Open BytaFont app, go to Swap mode > Emoji then tap on Restore BytaFont Backup to restore the default emoji font, respring after that
    2.
  2. Open Cydia to uninstall Emoji iOS 11 (font) and Emoji10 (tweak) of your iOS version
  3. Respring your device

Option #4: Manual

For iOS < 10 users, follow instructions here. Otherwise, continue reading.

Due to instability of iOS 10 jailbreak, the correct emoji font is meant to be installed manually. Here are steps:

Installation

  1. Download iOS 11 beta emoji font file: TTC
  2. Backup the default AppleColorEmoji@2x.ttc and AppleColorEmoji@2x.ccf in /System/Library/Fonts/Core of your device (Rename by adding .bak or simply move them to somewhere else)
  3. Copy the downloaded font into that directory, only the new AppleColorEmoji@2x.ttc (of size ~30 MB) will be inside /System/Library/Fonts/Core

Note: you can watch the video tutorial from here, just that get the font from the link provided above instead

Uninstallation

  1. Navigate to /System/Library/Fonts/Core, delete the new AppleColorEmoji@2x.ttc
  2. Restore the original fonts (AppleColorEmoji@2x.ttc of smaller size and AppleColorEmoji@2x.ccf), then respring your device

Troubleshooting

General concern

As far as I know, many people don't change the emoji font on their devices to the proper one after they install the tweaks. Since the original fonts won't contain any new emoji, users experience bugs at displaying emoji characters instead.

Issues with font

BytaFont (well, any version) is known to cause problems more than profits because it has to swap a new emoji font with the default one. If the swap fails or there is no enough disk space for the font, emoji keyboard is messed up.

To get around this problem, you have to see what happened inside the directory /System/Library/Fonts/Core (iOS 8.3+) or /System/Library/Fonts/Cache otherwise. The original font should still be there, but maybe with a wrong file extension.

  • For iOS 6.0 - 8.2, the exactly correct name of the emoji font is AppleColorEmoji@2x.ttf (retina devices) or AppleColorEmoji.ttf otherwise
  • For iOS 8.3 - 9.3, similar to the above but you will also have a file with the same name but with extension .ccf, which is in size larger than the ttf one
  • For iOS 10.0+, nothing is extraordinary than the new extension of the emoji font .ttc instead of .ttf, the manner is still the same

If you see something different, you are likely to have issues with the font. To restore default fonts, there are multiple ways:

  • Get BytaFont Backup fonts that should be in Cydia for you to recover lost fonts
  • /User/Library/Preferences/BytaFontBackup should keep all default fonts (provided that you use BytaFont), you can get the old emoji font from there
  • I do provide some system-default emoji fonts here that you may use
  • There are also some links to the font that you may find on the internet, or reddit /r/jailbreak
  • Download firmware or OTA update corresponding to your iOS version, extract files (decrypt if needed) and copy the original ones to the font directory

By installing the correct font, this should fix 90% of problems installing Emoji10 that new users usually encounter.

Emoji duplication bug

This problem can occur on iOS 8.3 and above with Emoji10 installation. The latest versions already prevent that from happening, but users may be able to see just because they used old versions. The thing is this bug persists even after uninstallation. To fix it, you can either:

  • Delete /User/Library/Preferences/com.apple.EmojiPreferences.plist and respring your device
  • Tap Reset emoji preferences in the settings page of EmojiFontManager that you are supposed to install just for resetting process
